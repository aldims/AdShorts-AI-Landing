# SEO-план AdShorts AI (по SEO_TASK.md)

Примечание: в `SEO_TASK.md` задан объём **10–30** SEO-страниц. По вашей задаче ("в итоге 50–70" + расширение под максимальный платёжеспособный трафик) ниже — **полный бэклог из 90 страниц** (70 базовых + 20 под конверсию/BOFU, без дублей по интенту). Для первого запуска достаточно опубликовать первые 10–30, остальное — масштабирование.

Общее для всех SEO-страниц:
- Если интент совпадает (синонимы/переформулировки одной боли) — делаем **одну** страницу и раскрываем формулировки внутри текста.
- Если интент разный (проблема / решение / техника) — делаем **разные** страницы.
- URL: читаемый, на русском, латиницей; отражает боль/технику.
- Title: 60–65 символов, содержит ключевой запрос.
- Description: 140–160 символов, описывает проблему и решение.
- H1 = поисковый запрос.
- Объём: 800–1 200 слов.
- Тон: простой, практичный, без маркетинговых штампов. Без переспама ключевых слов.
- CTA: 1–2 раза на странице (в первом экране и в конце).
  - Кнопка: **«Создать видео бесплатно»** (переход в Telegram-бот).
  - Текст под кнопкой: **«Откроется Telegram-бот — ролик можно собрать за минуту и сразу протестировать правки.»**
- Пункт из ТЗ «мягкая связь с продуктом»: на странице не пишем этой фразой — делаем как секцию **«Как быстрее тестировать правки»**.
- Ограничения: без внешних ссылок, без сложных форм, не перегружать CTA.
- Навигация: на лендинге есть блок «Гайд по YouTube Shorts» и отдельная страница со списком материалов `/shorts-guides/`.
